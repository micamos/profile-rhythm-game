# 自己紹介ページを「太鼓の達人」にしたら、スコアが高いほど恥ずかしい記事が読めるようになった話

## はじめに

プロダクトデザイナーとして自己紹介ページを作ることになったとき、ふと思った。

**「普通のポートフォリオ、誰も最後まで見なくない？」**

名前、経歴、実績……。整然と並んだ情報は正しいけれど、記憶には残らない。どうせ作るなら、訪問者が「なにこれ」と手を止めて、思わず触ってしまうものにしたい。

そこで生まれたのが、**音ゲーの選曲画面風の自己紹介ページ**だ。しかも、ただの見た目だけじゃない。STARTボタンを押すと本当に太鼓の達人風のリズムゲームが始まり、**スコアが高いほど、自分の恥ずかしい記事がアンロックされていく**という仕組みにした。

---

## コンセプト：「読まれる」ではなく「遊ばれる」自己紹介

自己紹介の目的は、自分を知ってもらうこと。でも「知ってもらう」のに、必ずしも一方的に情報を提示する必要はない。

ゲームという体験を通じて、訪問者が**能動的に**自分の記事にたどり着く。そのプロセス自体が、自分の人となりを伝えるコミュニケーションになる。そう考えた。

### なぜ「恥ずかしい順」なのか

記事のアンロック順は、ビュー数が多い無難な記事から始まり、スコアが上がるにつれて個人的でニッチな記事が解放されていく。

| スコア帯 | 解放される記事 | 恥ずかしさ |
|---------|-------------|---------|
| ~20% | 開発生産性の図解 | ★☆☆☆☆ |
| ~40% | Figma活用術 | ★★☆☆☆ |
| ~55% | 事業貢献の振り返り | ★★★☆☆ |
| ~70% | Designshipレポート | ★★★☆☆ |
| ~85% | ワーママインタビュー | ★★★★☆ |
| 85%~ | 5歳児とAI親子ゲーム開発 | ★★★★★ |

つまり、**頑張ってくれた人だけが、自分の素の部分を見られる**。これは単なるギミックではなく、「この人のことをもっと知りたい」というモチベーションをゲームのスコアという形で可視化している。

## デザイン：ニューモフィズム × 音ゲーUI

### なぜニューモフィズムか

音ゲーのUIといえば、ネオンカラーにグラデーション、派手なエフェクトが定番。でもあえて**ニューモフィズム**を選んだ。

理由はシンプルで、「触れそう」だから。

```css
.neu {
  background: #e0e5ec;
  border-radius: 14px;
  box-shadow: 6px 6px 12px #b8bec7, -6px -6px 12px #fff;
}
```

光と影だけで浮き上がるボタン、押し込むと凹むインタラクション。カラフルにしないことで、かえって「これ押せるの？」という好奇心を誘う。ゲームの世界観と自己紹介の落ち着いたトーンを両立できるデザインシステムだった。

### ドラムロール式の曲リスト

記事一覧は、縦スクロールのリストではなく**ドラムロール（円形ホイール）** で実装した。三角関数を使って、選択中のアイテムが手前に大きく飛び出し、離れるほど奥に小さくなっていく。

```javascript
const y = Math.sin(rad) * RADIUS;
const xPush = Math.cos(rad) * MAX_PUSH;
const scale = Math.max(0.55, 1 - normAngle / 100);
```

音ゲーの選曲画面そのものの操作感で、マウスホイールでもキーボードの矢印でも一曲ずつ滑らかに切り替わる。

## 実装：HTMLファイル1枚で完結

技術スタックは驚くほどシンプル。

- **HTML** 1ファイル（CSS・JSインライン）
- **Canvas API** でリズムゲーム描画
- **Web Audio API** でBGM・効果音をリアルタイム生成
- 外部ライブラリ・フレームワーク：**なし**

### Web Audio APIで祭り囃子を鳴らす

音源ファイルは一切使っていない。Web Audio APIのオシレーターとノイズバッファだけで、太鼓の達人っぽいBGMを生成している。

- **メロディ**: square波で祭り囃子風のフレーズ
- **ベースライン**: triangle波のマイナースケール
- **ドラム**: sine波のキック + ノイズバッファのハイハット
- **効果音**: ドン（周波数下降するsine波）、カッ（ハイパスフィルタ付きノイズ）

BPM 140の15秒間。短いけれど、ゲームとしてはちょうど気持ちいい長さになった。

### ノーツ生成のランダム性

毎回同じ譜面では飽きるので、ノーツはランダム生成。ただし完全ランダムではなく、最低間隔（280ms）を保ちつつ、25%の確率で連打パターンが挿入される。

```javascript
if (Math.random() < 0.25 && t + minGap * 3 < duration - 500) {
  for (let j = 0; j < 2; j++) {
    t += minGap;
    notes.push({ time: t, type: Math.random() < 0.5 ? 'don' : 'ka' });
  }
}
```

これにより「何度やっても新鮮」と「理不尽じゃない難易度」のバランスが取れている。

## 制作プロセス：AIとの共同制作

このページは**Claude Code（AIコーディングエージェント）との対話**で制作した。

制作の流れはこうだ：

1. 「自己紹介ページを作って」→ まずシンプルなHTMLが生成される
2. 「音ゲーの選曲画面風にして」→ レイアウトが変わる
3. 参考画像を渡して「これに合わせて」→ 2カラムレイアウトに再構築
4. 「ドラムロールにして」→ 三角関数ベースのホイール実装
5. 「note記事を入れて」→ 実際の記事データを取得・反映
6. 「太鼓の達人を入れて」→ Canvas + Web Audio APIでミニゲーム実装
7. 「スコアが高いと恥ずかしい記事が見れるようにして」→ アンロックシステム

一つ一つは小さな指示だが、積み重ねることで1枚のHTMLファイルが**インタラクティブな体験**に進化していった。

デザイナーとしての意思決定（コンセプト、レイアウト、インタラクションの方向性）は人間が行い、実装の速度と精度をAIが担う。この協業スタイルは、プロトタイピングの新しい形だと感じた。
